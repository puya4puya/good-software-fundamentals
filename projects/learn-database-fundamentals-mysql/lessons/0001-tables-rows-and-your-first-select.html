Tables, Rows, and Your First SELECT

A relational database stores data in tables — grids, a lot like a spreadsheet. This lesson gets you to the point where you can look at a table and pull exactly the data you want out of it with one line of SQL.

A table, precisely

A table has a fixed set of columns (the fields every record has) and any number of rows (one row = one record). Here's a tiny orders table — the kind of shape you'll recognize from an OpenMage store's sales_order table, simplified down to the essentials:

order_idcustomer_emailstatustotal
1001ana@example.comcomplete129.90
1002bogdan@example.compending59.00
1003ana@example.comcomplete245.50
1004cristina@example.comcanceled19.99

One more term you need now: order_id is the primary key — the column that uniquely identifies each row. No two rows share a primary key value; that's what makes it possible to point at "that one order" unambiguously.

Vocabulary check. Table = the whole grid. Row = one record (one order). Column = one field every row has (status, total, …). Primary key = the column guaranteeing each row is unique.

Your first SELECT

To ask a database for data, you write a SELECT statement. The shape is always: which columns, from which table, matching which condition.

SELECT order_id, status, total
FROM orders
WHERE status = 'complete';

Read it left to right: "give me the order_id, status, and total columns, from the orders table, but only rows where status equals 'complete'." Run against the table above, that returns rows 1001 and 1003 — 1002 and 1004 are filtered out because their status doesn't match.

Two things worth internalizing immediately:

  • * instead of column names means "all columns" — useful for exploring, but in real code naming the columns you actually want is clearer and cheaper.
  • Text values (like 'complete') go in single quotes. Numbers (like 129.90) don't.
Try it — what does this query return? 
SELECT customer_email, total
FROM orders
WHERE customer_email = 'ana@example.com';

Two rows — 1001 and 1003 — since those are the only rows where customer_email equals 'ana@example.com'. Each returned row shows only the two requested columns: customer_email and total.

Try it — write your own

Using the same orders table, write (on paper or in your head) a query that returns the order_id for every order that is not complete. Hint: SQL's "not equal" operator is <> or !=.

SELECT order_id
FROM orders
WHERE status <> 'complete';

Returns 1002 and 1004.
